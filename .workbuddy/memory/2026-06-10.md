## 2026-06-11 — 云端同步存储（多设备互通）

### 需求
用户需要编辑内容在线保存，不同设备/浏览器之间互通同步。

### 方案调研
- jsonblob.com: GET 支持 CORS（简单请求），但 PUT 触发浏览器 CORS 预检被拦截
- corsproxy.io / allorigins.win: 公开 CORS 代理均被 Cloudflare 拦截或不稳定
- npoint.io: 支持 CORS 但仅限 GET/POST/OPTIONS
- **最终方案**: 自建 Node.js Express 后端，JSON 文件存储

### Node.js 后端架构
- **位置**: `server/` 目录
- **文件**: 
  - `server.js` — Express 服务器，端口 3000
  - `package.json` — 依赖 express
  - `public/index.html` — 前端页面（API 调用改为相对路径）
  - `data/content.json` — 持久化存储
- **API**:
  - `GET /api/content` — 公开读取（无需密码）
  - `POST /api/content` — 受保护写入（需密码 admin123）
  - `DELETE /api/content` — 受保护清空（需密码）
  - `GET /api/health` — 健康检查
- **CORS**: 全开放（`Access-Control-Allow-Origin: *`）

### 前端适配（server/public/index.html）
- `CLOUD_BLOB_URL` → `API_BASE = '/api/content'`
- `syncToCloud()`: PUT → POST，附带 `password` 字段
- `loadContentState()`: 从 `/api/content` GET 读取
- `resetContent()`: 发送 DELETE 清空云端
- 导航栏同步状态指示器（☁️ 已同步 / 🔄 同步中... / ⚠️ 同步失败 / 📴 本地模式）
- 管理面板新增「手动同步到云端」按钮

### 启动脚本
- `server/start.bat` — 本地启动（仅 localhost:3000）
- `server/start-public.bat` — 公网启动（localtunnel 生成公开 URL）

### 同步流程
1. 页面加载 → GET `/api/content` → 应用云端内容 → localStorage 缓存
2. 管理员编辑 → blur 触发保存 → localStorage 即时保存 → 800ms debounce → POST `/api/content`
3. 模块增删 → 立即触发 `saveContentState()` → 云端同步
4. 恢复默认 → DELETE `/api/content` → 清空 localStorage → 刷新页面
